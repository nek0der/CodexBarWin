# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.x.x   | :white_check_mark: |

> **Note**: Once version 1.0.0 is released, this table will be updated to reflect the supported versions.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do NOT** create a public GitHub issue
2. Email the maintainer directly or use GitHub's private vulnerability reporting feature
3. Include detailed information about the vulnerability:
   - Description of the issue
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes (if applicable)

## Response Timeline

- **Initial Response**: Within 48 hours
- **Status Update**: Within 7 days
- **Resolution Target**: Within 30 days (depending on severity)

## Security Best Practices for Users

1. Always download releases from the official GitHub repository
2. Verify file checksums when available
3. Keep the application updated to the latest version
4. Report any suspicious behavior

## Scope

This security policy covers:

- CodexBarWin application
- Official releases and builds

Third-party dependencies are covered by their respective security policies.
